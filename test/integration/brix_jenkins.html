Young Suk's description of the above tasks:

I have setup Jenkins jobs (so far DEV environment) for deploying.

The deployment involves

  1. In build server: pull master branch from repo, run junit, when success:
  2. In remote actual server: stop application, pulling from repo, start application

NOTE:

  1. Nothing will be shown in the browser (unless there is access error), just wait few secs and see if changes were applied.
  2. The dev-images will not trigger service restart
  3. You can put any note in the last parameter “cause”, e.g. &cause=Super+Neat+change
    If there is no note, put your initials.
    In the example above I put "YourInitials"
    This page doesn't let you set what the "cause" should be. A future enhancement will add a field that will be sent as the cause.
